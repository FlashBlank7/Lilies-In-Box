# Playtest Notes

## 2026-06-25：Workflow Balance Pass 1

### 怎么体验

- 跑 `python3 tools/verify_chapter.py` 确认结构和文本约束。
- 用 Godot headless smoke 跑完整流程，覆盖序章、回声台阶、Chapter 1-3 和 Finale。
- 枚举每个目标的推荐顺序和常见漏节点顺序，检查成功余量、失败原因和下一步提示。

### 发现

- Chapter 1 的前三个推荐 workflow 太接近阈值，读起来像精确算分，不像“她理解得更稳了一点”。
- `Wait` 的收益偏弱，第二章和 Finale 的等待节点像一个硬性钥匙，而不是有体感的安静。
- 部分 UI 仍然太工程化：`workflow 抽屉`、`trace`、目标旁边的英文 kind 标签会让梦醒一下。
- 新章节短篇骨架能跑通，但美术还只是气氛草图，后续需要逐章加厚。

### 已调整

- 降低 Chapter 1 前三关的置信阈值，让正确思路有余量，漏证据流程仍然失败。
- 提高 `Wait` 的置信收益，让等待更像真实地让房间安静下来。
- 提高没有视觉证据时 `Compare` 的收益，让第三章的“听见后比较”更自然。
- 把玩家可见的部分 `trace/workflow` 文案改成“回声/抽屉/顺序”。
- 把目标旁边的英文 kind 标签改成中文意象名。

### 下一轮优先看

- 实机窗口里逐章看画面：第二章无字积木、第三章纸张、Finale 空白房间是否一眼能分辨。
- 检查抽屉 UI 是否太高、是否遮挡莉莉丝或目标。
- 给第二章失败时的噪声压力增加更明显但不焦虑的视觉层次。
- 让 Chapter 3 的“旧回声被记住”不只是文案，而是影响目标卡或场景纸片。

## 2026-06-25：Visual Feedback Pass 2

### 怎么体验

- 复查章节脚本和抽屉布局，重点看“第一眼能不能分辨章节”和“失败有没有场景反馈”。
- 继续使用完整 Godot smoke 证明所有推荐 workflow 仍能跑通。

### 发现

- Chapter 2 的主题还停留在文案里，背景里的无字积木感不够。
- Chapter 3 已经有“旧回声被记住”的台词，但画面不会留下任何痕迹。
- Finale 成功后缺少一个温柔的视觉结算，玩家只看到普通完成流程。
- 抽屉的 10 个节点用 4 列时会占三行，面板偏高，容易遮住下半屏。

### 已调整

- Chapter 2 背景增加淡色无字积木影子，失败时目标旁生成柔和噪声碎片。
- Chapter 3 失败后会在场景左侧留下小纸片回声。
- Finale 成功后在空白条附近浮现“莉莉丝”。
- 抽屉改为 5 列节点布局，面板高度降低，减少遮挡。

### 下一轮优先看

- 用截图或窗口实测每章第一屏，确认 Chapter 2/3/Finale 的区分度足够。
- 让 Chapter 3 的纸片回声进入目标卡或 trace 区，不只留在背景。
- 检查抽屉压缩后文字是否仍然舒适可读。

## 2026-06-25：Visual Screenshot Pass 3

### 怎么体验

- 新增 `tools/visual_playtest_capture.gd`，用 1280x720 固定视角自动截取 Chapter 2、Chapter 3、Chapter 3 失败回声和 Finale。
- 视觉截图需要普通 Godot 渲染器运行：`Godot --path . --script tools/visual_playtest_capture.gd`。headless 的 dummy renderer 不能读 viewport 贴图。
- 截图输出到 `/tmp/lilies_playtest_captures`，用于每轮对比第一眼画面和抽屉遮挡。

### 发现

- 章节标题和第一句诗性提示如果留在截图中央，会盖住目标和右侧纸条，无法判断真实可玩状态。
- Chapter 2、Chapter 3、Finale 的机制已经不同，但背景仍太像同一座教堂。
- 玩家可见文案里还残留 `workflow`、`trace`、`终端动作` 这类过直的词。

### 已调整

- 自动截图会等待章节浮层退场后再拍，第一屏更接近玩家实际操作时的读图状态。
- Chapter 2 增加悬浮无字积木轮廓和拒绝线，让“不拿、不伸手”的主题进入背景。
- Chapter 3 增加更明显的信封、折线和未写完的纸痕，失败回声纸片更亮。
- Finale 用更深的空房间遮罩和中央空白条，把目标从“又一扇门”改成“留下名字的位置”。
- 玩家可见的技术词继续软化为“顺序 / 回声 / 收尾动作”，代码内部仍保留 workflow 命名以支持后续系统扩展。

### 下一轮优先看

- 抽屉打开时仍会遮住莉莉丝左下活动空间，后续可以改成可收起的窄抽屉或右侧纵向纸条。
- 右侧目标卡信息量偏大，后续可以拆成“当前目标”和“上次回声”两个更轻的纸片。
- 第二章的“代价”目前主要是数值和噪声层，后续需要让朋友变暗的视觉状态更持续。

## 2026-06-25：Drawer And Cost Pass 4

### 怎么体验

- 用 `tools/visual_playtest_capture.gd` 对比 Chapter 2 抽屉打开、Chapter 2 代价失败、Finale 抽屉打开三张关键图。
- 重点看莉莉丝是否还被 UI 盖住、目标卡是否还能快速读懂、第二章“不要马上推”是否有可见后果。

### 发现

- 旧抽屉虽然变矮了，但仍覆盖莉莉丝的站位，玩家会失去“小女孩在旁边犹豫”的画面中心。
- 右侧目标卡的阈值、风险上限、代价上限一起显示时太像调参表，会削弱梦境表达。
- 2-3 的“代价”原本会被缺少等待证据挡住，玩家很难真正看到“太亮的门会让朋友变暗”。

### 已调整

- 抽屉改成莉莉丝右侧的两栏纸夹：左边放积木和当前顺序，右边放朋友回声和下一次方向。
- 右侧目标卡移除裸露阈值行，只保留目标、需要听见的证据、收尾动作、上次回声和下一次建议。
- 2-3 改成先能理解门，再因为没等待而代价太重；失败后场景里留下一个变暗的朋友影子。
- 截图工具新增 `chapter2_cost_shadow.png`，后续可以持续检查代价反馈是否清楚。

### 下一轮优先看

- 抽屉按钮文字仍偏密，后续可以给节点加短中文意象名或小图标。
- 右侧目标卡和顶部目标行有一点重复，后续可以进一步把顶部目标行变成更轻的章节状态。
- 第二章暗影反馈已经可见，但还没有影响后续运行的情绪节奏，之后可以让 Wait/Refuse 逐渐擦亮它。
