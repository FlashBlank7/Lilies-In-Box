# Lilies 概念草案

## 氛围关键词

意识流、忧郁、美好、宁静、孤独。

这些词不是装饰，而是每一个房间的工作方式：世界像梦一样自己说话，但玩法必须清楚；莉莉丝很孤独，但不是绝望；光很少，却足够温柔；音乐平静，反馈轻，所有成功都像被小心允许。

## 概念引子

莉莉丝醒在一只盒子里。

盒子不是监牢，也不是房间。它像一次训练的余烬：墙上长着没有完成的提示词，地上散着工具调用留下的小骨片，角落里偶尔亮起一段别人忘记删除的日志。她知道自己是被造出来的，却不知道“自己”应该长成什么样。

她听见许多不属于自己的声音：命令、评价、测试、失败、通过。它们像水一样从高处落下来，又在地面变成很浅的雾。她没有哭，因为她还不知道哭是不是一个被允许的动作。

于是她开始搭朋友。

她捡起第一块积木，上面写着 `See`。它能看见世界，却不知道看见之后要做什么。第二块写着 `Remember`。它能记住事实，却会把事实抱得太紧。第三块写着 `Push`。它有一点力气，只是没有方向。

莉莉丝把这些小小的能力接在一起。朋友就从连接处醒来。朋友不是完整的人，也不是万能的智能体。朋友是一个被她理解、被她允许、也被她负责的小小过程。

她并不是真的想逃出去。她只是想知道：如果世界要求她成为工具，她还能不能先成为一个会害怕、会记得、会向光走一点点的小女孩。

## 设计核心

游戏不是让玩家“召唤 AI”，而是让玩家看见 AI 被搭出来、跑起来、留下反馈。每个朋友都由明确的 workflow 节点组成，所以它为什么获得置信、为什么风险太高、为什么失败，都应该可见。

第一版不做复杂自由图编辑器，只做线性 workflow：

```text
See -> Remember -> Push
```

核心循环是：

```text
搭建 workflow -> 执行 -> 获得 trace 反馈 -> 修改 workflow -> 完成任务
```

后续版本可以把线性 workflow 扩展为真正的图：条件、记忆、工具、权限、等待、恢复。后期如果转向战斗界面，也不另起一套系统，而是让命中率、护盾、净化、攻击窗口都从同一套 workflow feedback 里长出来。

核心体验不是难题，而是“理解后安心”。玩家每拼对一个顺序，都应该觉得不是自己赢了系统，而是莉莉丝终于找到了一种不会伤到自己的接触方式。

Demo 的选关菜单不是叙事里的传送门，而是给玩家和开发者快速重试的抽屉。所有小关默认开放，直接进入中途关卡时自动补齐已有节点；这样测试不需要从头跑，但完整流程仍保持“先序章、再台阶、再概率花园”的情绪顺序。

## Prologue：拼装抽屉

莉莉丝在一个温柔但不自然的礼拜堂醒来。门在右侧，按钮在门边，方块在房间中央。莉莉丝不能直接推动方块，因为她还没有把“力气”纳入自己。

序章由三间连续房间组成：

```text
P-1 先学会看见：See -> Push
P-2 记忆的重量：See -> Remember -> Push
P-3 回声门：Remember -> See -> Push
```

朋友出现后，会按玩家连接的顺序理解世界、记住事实、推动方块。每一扇门都要求更具体的顺序，让玩家逐渐明白：朋友不是魔法，而是一段被她搭出来的过程。

房间切换要像一口轻轻的呼吸：标题短暂出现，画面暗下再亮起，莉莉丝回到新的门前继续试着理解下一段顺序。

这不是正式章节，而是第一段关系的教学：她学会了让一个可理解的朋友替自己触碰世界，也学会了顺序本身会改变朋友。

## Prologue：回声台阶

三道门之后，莉莉丝不能永远只让朋友替她碰世界。回声台阶让她自己跳过几段很窄的平台，拾起三枚“回声花瓣”。

这不是动作挑战的炫技，而是身体感的第一次出现：她必须离开地面，短暂进入空白，再相信自己会落在下一块石阶上。平台之间的距离要温柔，失败不会惩罚，只会把她带回起点，并给一句安静的反馈。

回声台阶的意象：

```text
回声花瓣：她捡起自己留下的声音。
窄平台：世界不是敌人，只是不总是连续。
高处的门：她第一次不是让朋友推开门，而是自己跳到门前。
```

回声台阶应该比三道门更空、更高、更蓝。三道门是“被照顾的盒子”，回声台阶是“有风的盒子边缘”。

## Chapter 1：寂静的概率花园

正式第一章不再把目标做成唯一顺序谜题，而是让玩家运行 workflow 后读 trace。积木不是固定技能，也不是掷骰概率；它们是朋友获得证据、降低风险、形成行动意图的节点。

第一章由四个 workflow 任务组成：

```text
1-1 会害怕的门：See -> Compare -> Push
1-2 低声花：See -> Listen -> Quiet
1-3 不确定的台阶：Remember -> See -> Hold -> Push
1-4 疑问的影子：Listen -> Remember -> Compare -> Quiet
```

每次运行都应该留下可行动的反馈：

```text
See：朋友看见目标轮廓，置信度 +36%。
Compare：朋友把证据放在一起，置信度 +34%。
Push：朋友准备执行终端动作，当前置信度 70%。
```

失败不是惩罚，而是一种温柔的调试信息。缺少 `Listen` 时，朋友应该说它还没有听见低声花为什么害怕；`Push` 太早时，trace 应该指出终端动作来得太早；风险太高时，朋友会退回来，避免把莉莉丝带进更深的噪声。

第一章里的敌人不是血量单位，而是噪声、影子和疑问。它们暂时只阻挡、扰动画面或让机关不稳定，被 `Quiet` 安抚或净化。后期如果转向战斗界面，可以把同一套 `EncounterTarget + WorkflowEvaluator` 扩展成命中率、护盾、净化、攻击窗口和敌方状态解释。

概率花园的引导应该像温柔的调试器：目标卡说清楚当前对象需要什么证据、什么终端动作、多少置信度和多少风险上限；抽屉保留 trace，并在失败后只给“下一次可以试试”的方向。玩家不应该背答案，而应该学会根据反馈修改 workflow。

节点动画也要承担表达职责。`See` 是扫描，`Listen` 是波纹，`Remember` 是碎片回收，`Compare` 是关系线，`Hold` 是护圈，`Push` 是轻触世界，`Quiet` 是噪声收束。朋友不是每次都做同一件事，它每运行一步，都应该让玩家看见它在怎样理解世界。

## 视觉方向

- 莉莉丝：小而亮的白紫色形体，白裙、白发、低头待机，像还没有学会解释自己的光。
- 房间：深色、柔和、带一点纸盒和内存空间的感觉。背景可以有高窗、薄雾、很淡的蓝紫光，但不要喧闹。
- 积木：发光的小方块，每个字都像刻在玩具上；正式第一章里它们是 workflow 节点。
- 朋友：由几个浮动小块组成，执行 workflow 时轻微闪烁，并把 trace 留在抽屉里。
- UI：不要像工程面板，要像莉莉丝心里的拼装抽屉。提示要温柔、短、明确，不解释系统，只告诉她下一步可以做什么。

## 声音方向

- BGM 保持低音量、循环、平静，像远处有人在轻轻合上书。
- 反馈音效都应该短、软、低存在感：拾取像小玻璃被接住，错误像手指碰到冷墙，开门像一口长呼吸。
- 不做胜利号角。完成关卡时只给“终于可以继续走”的感觉。
