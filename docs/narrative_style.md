# Narrative Style Guide

## 原则

游戏文本要像梦里的纸条，而不是作者解释主题。玩家可以不知道底层隐喻，也要能靠提示玩下去；但提示只能说“下一次可以怎样轻一点”，不能说“这个系统象征什么”。

## 禁用词

游戏内文本禁止出现：

```text
agent
验收
发布
平台
自动创建智能体
创建自己的 agent
为什么被创建
```

这些词可以存在于开发讨论和代码注释，但不能出现在玩家会看到的字幕、目标卡、纸条、菜单剧情或结尾文案里。

## 推荐写法

- 用物件承载意义：抽屉、纸、无字积木、铃、门缝、花、影子、回声。
- 用短句和留白，不用大段说明。
- 失败提示要可行动，但仍然温柔：告诉玩家缺了听见、等待、比较、稳住，而不是说“答案错误”。
- 结尾不总结主题，只展示变化后的物件。

## 示例

不要写：

```text
莉莉丝正在寻找自己的验收标准。
```

改成：

```text
抽屉里没有最后一张纸。
她把无字积木握了一会儿。
它没有变亮，也没有离开。
```

不要写：

```text
这个 agent 不应该被发布。
```

改成：

```text
朋友已经走到门前。
可莉莉丝看见它的光暗了一下。
她把手收回来。
```

不要写：

```text
请使用拒绝节点完成任务。
```

改成：

```text
它不一定要被拿走。
先看见它，再把手收回。
```
